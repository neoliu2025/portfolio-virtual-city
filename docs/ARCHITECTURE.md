## 前端架构

### 核心模块

1. **3D 场景层** (Three.js)
   - 城市地图渲染
   - 建筑模型管理
   - 摄像机系统
   - 光影效果

2. **交互层**
   - 键盘/鼠标输入
   - 触摸手势识别
   - 碰撞检测
   - 选择系统

3. **状态管理** (Pinia)
   - 用户状态
   - 城市状态
   - UI 状态
   - 临时数据

4. **业务逻辑** (Composables)
   - 摄像机控制
   - 动画管理
   - 音频播放
   - 数据请求

5. **UI 组件** (Vue 3)
   - 建筑内部 UI
   - 模态框
   - 按钮 & 表单
   - 通知中心

## 后端架构

### API 分层

1. **路由层** (Routes)
   - 请求分发
   - 路由定义

2. **中间件层** (Middleware)
   - 认证验证
   - 请求日志
   - 错误捕获
   - CORS 处理

3. **控制器层** (Controllers)
   - 业务逻辑
   - 参数验证
   - 错误处理

4. **模型层** (Models)
   - 数据模式
   - 验证规则
   - 关系定义

5. **数据层** (Database)
   - 数据持久化
   - 查询优化
   - 缓存策略

## 关键技术选型

| 层级 | 技术 | 原因 |
|------|------|------|
| **前端框架** | Vue 3 | 学习曲线温和，性能好 |
| **3D 引擎** | Three.js | 生态成熟，文档完善 |
| **动画库** | GSAP | 性能出色，易用性强 |
| **样式** | Tailwind CSS | 快速开发，高度可定制 |
| **打包工具** | Vite | 构建速度快，开发体验好 |
| **后端框架** | Express.js | 轻量级，社区活跃 |
| **数据库** | MongoDB | 灵活的模式，易于扩展 |
| **类型** | TypeScript | 代码更安全，开发效率高 |

## 数据流

```
用户交互 (WASD/鼠标/触摸)
    ↓
输入处理 (useInteraction composable)
    ↓
Pinia 状态更新
    ↓
组件重新渲染
    ↓
Three.js 场景更新
    ↓
画面重新渲染

用户操作 (点击进入建筑/提交表单)
    ↓
数据验证
    ↓
API 请求 (axios)
    ↓
后端处理
    ↓
数据库操作
    ↓
API 响应
    ↓
Pinia 状态更新
    ↓
UI 更新
```
